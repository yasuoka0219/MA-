# メール送信方法

メールを送る際に**送信相手を絞り込んだり選択できる**ことを前提に、現在の送信方法と、今後のあるべき流れをまとめています。

---

## 現在できる送信方法

### 1. イベントの参加者を選んで送る（送信先を選択できる唯一の方法）

**手順**

1. ログインし、**イベント** メニューを開く。
2. 送信したい **イベント** をクリックする。
3. 「登録済み参加者」の表で、**送りたい参加者にチェック** を入れる（全選択も可）。
4. **テンプレート** をプルダウンで選ぶ。
5. **「選択した参加者にメール送信」** をクリックし、確認ダイアログで送信。

**絞り込み・選択**

- 送信先は **そのイベントの参加者のみ**。
- 参加者一覧の **チェックボックスで誰に送るかを選択** できる。
- イベントに参加登録していないリードには、この画面からは送れない。

**前提**

- 承認済みの **メールテンプレート** が1つ以上あること。
- イベントに参加者が登録されていること。

**登録済み参加者になるには**

「登録済み参加者」は、**管理者がイベント詳細画面から手動でリードを追加**すると増えます。自動登録（例：申込フォームからの登録）はありません。

1. **イベント** メニューで該当イベントを開く。
2. **「参加者管理」** ブロック内の検索フォームで、追加したいリードを探す。
   - **キーワード検索** … 名前・メール・ID で検索。
   - **卒業年度** … 年度で絞り込み。
   - **同意済のみ** … 同意済みリードだけ表示するか選べる。
3. **「検索」** を押すと、条件に合うリードが「検索結果」の表に表示される。
4. 参加者にしたいリードを追加する。
   - **1件ずつ** … 行の **＋** ボタンをクリック。
   - **複数件まとめて** … チェックボックスで選び、**「選択したリードを一括追加」** をクリック。
5. 追加されたリードが **「登録済み参加者」** の表に表示され、ここからメール送信の対象を選べるようになる。

※ すでにこのイベントに登録済みのリードは検索結果に出ません。参加者を外したい場合は、登録済み参加者表の **削除** で解除できます。

---

### 2. シナリオによる自動送信（条件で送信先が決まる）

**手順**

1. **テンプレート** を作成し、承認する。
2. **シナリオ** で「トリガー（例: イベント参加・リード登録）」と「そのテンプレート」を設定する。
3. トリガーが発生すると、条件を満たすリードに **自動で** メールが送られる（送信枠 9:00〜20:00 JST、スケジューラが実行）。

**絞り込み・選択**

- 送信先は **シナリオのトリガーとセグメント条件** で決まる。
- 「この人だけ」「この一覧で選んだ人だけ」のように **手動で選ぶことはできない**。
- **送信ログ** で誰に送ったかは確認できる。

---

### 3. 送信ログからの再送（失敗・ブロック時）

**手順**

1. **送信ログ** を開き、対象のログの **詳細** に入る。
2. ステータスが「失敗」または「ブロック」の場合、**「再送」** で同じ内容を再度送信できる。

**絞り込み・選択**

- 再送は **1件の送信ログに対して1通** のみ。送信先の追加選択や絞り込みはない。

---

### 4. パスワード再設定メール（システム用）

ログイン画面の「パスワードを忘れた」から、**登録ユーザーのメールアドレス** に再設定用メールを送る。MA配信の「送信先を選ぶ」用途ではありません。

---

## 現在の制限（送信先の絞り込み・選択）

| やりたいこと                         | 現状 |
|--------------------------------------|------|
| リード一覧で条件を絞って送りたい     | 不可。イベント経由かシナリオのみ。 |
| 高温度リード（Hot/Warm）に送りたい   | 不可。一覧から送信する入口なし。   |
| このリード1人にだけ送りたい          | 不可。リード詳細に「メールを送る」なし。 |
| 複数リードを選んで一括送りたい       | イベント参加者に限り可能。それ以外は不可。 |

そのため、**「送る相手を絞り込んだり選択する」** は、今のところ **イベントの参加者を選んで送る** 場合にしかできません。

---

## 今後のあるべき流れ（送信先の絞り込み・選択を入れた送信方法）

メール送信を **「送信先の絞り込み・選択 → テンプレート選択 → 送信」** で統一すると分かりやすくなります。

### 共通の流れ（案）

1. **送信先を決める**
   - **リード一覧** … 検索・フィルタ（卒業年度・興味・配信状況など）で絞り込み、チェックで送信対象を選択。
   - **高温度リード一覧** … Hot/Warm/Cold や条件で絞り込み、チェックで送信対象を選択。
   - **リード詳細** … 「このリードにメールを送る」で1件だけ送信。
   - **イベント参加者** … 従来どおり、参加者一覧でチェックして選択（現状のまま）。
2. **テンプレートを選ぶ**  
   承認済みメールテンプレートから1つ選択。
3. **プレビュー・確認**  
   件名・本文（差し込み反映）と送信対象件数（または宛先）を表示。
4. **送信**  
   確認後、送信。本番以外の環境ではリダイレクト先を表示。

### 画面ごとの送信入口（案）

| 画面             | 送信の入口（案） |
|------------------|------------------|
| リード一覧       | 検索・フィルタで絞り込み → チェックで選択 → 「選択したリードにメール送信」。 |
| 高温度リード一覧 | 温度・条件で絞り込み → チェックで選択 → 「選択したリードにメール送信」。 |
| リード詳細       | 「このリードにメールを送る」→ テンプレート選択 → プレビュー → 送信。 |
| イベント詳細     | 現状どおり。参加者をチェック → テンプレート選択 → 「選択した参加者にメール送信」。 |

このようにすると、**どの画面からでも「誰に送るか」を絞り込み・選択してからメールを送る** という一貫した送信方法にできます。

---

## 「リードを絞り込んで送る」専用フローの流れ（案）

イベントを経由せず、**リード一覧や高温度リード一覧から直接「絞り込み → 選択 → 送信」** する場合の流れです。※未実装の案です。

### パターンA：リード一覧から送る

| ステップ | 画面・操作 | ユーザーがやること |
|----------|------------|--------------------|
| **1. 入口** | サイドメニュー **リード一覧** を開く | いつものリード一覧を開く。 |
| **2. 絞り込み** | 既存の検索・フィルタを使う | 検索（メール/名前/学校名）、卒業年度、興味関心、配信状況（配信中のみ など）で対象を絞る。検索を実行すると表が更新される。 |
| **3. 選択** | 表にチェックボックス列を表示 | 送りたいリードにチェックを入れる。「全選択」も可。選択件数が表示される。 |
| **4. 送信開始** | 「選択したリードにメール送信」ボタン | チェックが1件以上あるときだけボタンが有効。クリックすると「メール送信」画面（またはモーダル）に進む。 |
| **5. テンプレート選択** | 送信先サマリ + テンプレート選択 | 「送信先：○件」と表示。承認済みメールテンプレートから1つ選ぶ。 |
| **6. 確認** | プレビュー・確認 | 件名（差し込み例）、本文プレビュー（1件目で代表表示など）、送信件数。本番以外なら「リダイレクト先：○○」を表示。 |
| **7. 送信** | 送信実行 | 「送信する」で実行。結果画面で「○件送信しました（○件スキップ）」などを表示。 |

※ 配信停止中・未同意・メール未登録のリードは、あらかじめ選択肢から外すか、送信時にスキップして件数に含めない。

---

### パターンB：高温度リード一覧から送る

| ステップ | 画面・操作 | ユーザーがやること |
|----------|------------|--------------------|
| **1. 入口** | サイドメニュー **高温度リード** を開く | 高温度リード一覧を開く。 |
| **2. 絞り込み** | 既存のフィルタを使う | 温度（Hot/Warm/Cold）、卒業年度、直近反応（日数）、PV数（7日）、検索で絞る。 |
| **3. 選択** | 表にチェックボックスを表示 | 送りたいリードにチェック。「全選択」も可。 |
| **4 ～ 7** | 上記パターンAと同じ | 「選択したリードにメール送信」→ テンプレート選択 → 確認 → 送信。 |

---

### パターンC：専用「メール送信」画面でまとめてやる（オプション）

「リード一覧」「高温度リード」のどちらからでも、「メール送信」という**専用画面を1つ**用意する案です。

| ステップ | 画面・操作 | 備考 |
|----------|------------|------|
| **1. 入口** | サイドメニュー **メール送信**（新規） | 最初から「送信先を選んでメールを送る」目的の画面。 |
| **2. 送信先を選ぶ** | タブまたはリンクで「リード一覧で選ぶ」「高温度リードで選ぶ」を切り替え | それぞれリード一覧／高温度リードと同じ絞り込み・チェックUIをこの画面に埋め込む（または同じコンポーネントを再利用）。 |
| **3 ～ 7** | 上記と同様 | 選択 → テンプレート → 確認 → 送信。 |

※ パターンCにすると「メールを送りたいときはここ」がはっきりしますが、実装量は増えます。まずは **パターンA・B（各一覧に「選択してメール送信」を足す）** だけでも、シンプルに「リードを絞り込んで送る」は実現できます。

---

### イベント参加者への送信（従来どおり）

- **イベント** メニュー → イベント詳細 → 「登録済み参加者」でチェック → テンプレート選択 → 「選択した参加者にメール送信」。
- 上記の「リードを絞り込んで送る」フローとは別の入口のまま残し、使い分けできるようにする。

---

## 送信時の注意（共通）

- **承認済み** のメールテンプレートだけ送信可能。
- **配信停止中・未同意・メール未登録** のリードは送信対象外（スキップされるか、選択できないようにする）。
- 本番以外の環境では、設定により **全メールがテスト用アドレスにリダイレクト** される場合があります。送信ログで実際の送信先を確認してください。
